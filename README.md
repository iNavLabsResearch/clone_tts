# milli — Finetune Indic-Mio (MioTTS) on Gujarati Vaani

Finetune [`SPRINGLab/Indic-Mio`](https://huggingface.co/SPRINGLab/Indic-Mio) to learn new
Gujarati voices, prosody, and pauses, using the Gujarati subset of
[`ARTPARK-IISc/Vaani`](https://huggingface.co/datasets/ARTPARK-IISc/Vaani). Built for
**Kaggle 2× T4** (distributed data-prep and training).

## How the model actually works (important)
Indic-Mio is a **`Qwen3ForCausalLM`** that emits **MioCodec audio tokens** (`<|s_0|>…<|s_12799|>`,
25 Hz). The prompt is plain ChatML: the `user` turn is the text, the `assistant` turn is the
`<|s_N|>` token string. **Speaker timbre is NOT an LLM token** — it is a continuous codec
`global_embedding` applied at decode time. So:
- **Finetuning the LLM** teaches Gujarati pronunciation / prosody / `<pause>`.
- **A voice / "speaker ID"** = one `global_embedding` preset per speaker (built by `speakers.py`),
  used at inference for zero-shot cloning.

## T4 notes
fp16 (no bf16), SDPA attention (no FlashAttention-2), LoRA via PEFT, 8-bit AdamW, gradient
checkpointing, DDP via Accelerate (Unsloth is single-GPU only).

## Secrets (rotate the tokens you pasted!)
The tokens shared in chat are exposed and look revoked — **rotate them in HF/W&B settings.**
Set them in the environment (or `secrets.env`, gitignored):
```bash
cp secrets.env.example secrets.env   # then fill HF_TOKEN, WANDB_API_KEY
```
`HF_TOKEN` must have **accepted access** to the gated Vaani dataset and write access for pushing
`milli_guj_dataset_artpark`.

## Run order
```bash
# 1. install
bash setup.sh

# 2. dry-run dataset prep (no push) — sanity check filtering/ASR/encoding
python db_prep.py --config Gujarat_Valsad --limit 8 --no-push

# 3. build the dataset across both T4s (all 4 Gujarat configs)
bash scripts/run_db_prep_2gpu.sh
#   ...pushes shards to <your-hf-user>/milli_guj_dataset_artpark

# 4. build per-speaker voice presets
python speakers.py

# 5. finetune on 2× T4
bash scripts/launch_train_2gpu.sh

# 6. inference
python inference.py --text "કેમ છો?" --speaker SID_187999 \
    --adapter outputs/indicmio-guj --out out.wav
```

## Config
Everything is driven by [`config.json`](config.json): model/codec/ASR ids, dataset configs and the
**Gujarati-only filter**, LoRA + optimizer + scheduler hyperparameters, and W&B project
(`milli-indic-mio-gujarati`). Set `dataset.output.namespace` to your HF username (or rely on the
token's namespace).

## Processed dataset columns (`milli_guj_dataset_artpark`)
`id, speakerID, gender, district, language, duration, text, transcript, transcript_source,
speech_tokens (list[int]), num_speech_tokens, global_embedding (list[float])` — plus `audio` only
if `dataset.output.keep_audio` is true. Each prep shard is pushed as its own dataset *config*
(`<VaaniConfig>_w<worker>_s<shard>`); training concatenates them all.

## Caveats
- Vaani Gujarati rows have **no transcripts** → generated by IndicConformer ASR (some noise expected).
- "Pauses" learned are `<pause>` tags inserted from silence detection; `[breathing]`/`<static>`
  cannot be recovered by ASR.
- Real runs require CUDA (the code is device-aware but training/encoding need a GPU).
